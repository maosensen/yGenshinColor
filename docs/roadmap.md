# yGenshinColor — 项目构想与开发路线

> 状态:v1(2026-07-22 拍板)。本文档是产品与技术方向的单一事实来源;
> 实现层面的规范继续遵循 `AGENTS.md` 与 `docs/theming.md`。

## 一句话定位

**跟着原神学配色** — 从提瓦特场景截图中提取大师级色卡与渐变,配上
"为什么和谐"的注解,做成可复制、可导出、可分享的开源配色学习站。

非官方粉丝项目(Fan-made, non-commercial),永久免费开源。

## 三条产品原则

1. **学,不只是素材。** 每个场景必须回答"为什么好看":色相关系、
   明度节奏、饱和度分布。没有注解的场景不上线。素材站烂大街,
   教育内容才有传播属性。
2. **UI 即产品。** 展示层的说服力就是项目的说服力。内容定义界面
   颜色(场景色卡实时染页面),而不是界面框住内容。
3. **作品即传播。** 每个页面天生是一张可截图转发的海报;OG 卡片、
   导出格式、渐变动画都为"被分享"设计。

## 已拍板的决策

### 品牌与视觉

| 决策点 | 结论 |
|---|---|
| 站点工作名 | **Teyvat Palette**(中文副题「提瓦特色卡·跟着原神学配色」;M0 可再改,仓库名不变) |
| 基底氛围 | **深灰蓝 + 微噪点纹理**,约 `oklch(0.16 0.02 250)` 一族。纯黑太硬,所有色卡都"炸"反而没层次;深灰蓝是展示氛围站的成熟解(Figma 社区 / ReactBits),也贴提瓦特夜空气质 |
| 明暗模式 | **默认深色**。浅色不做成偏好开关,做成「白底预览」功能:看色卡在浅色设计里的效果 |
| 布局 | **顶部导航的展示型布局**,替换现有 dashboard sidebar shell(画廊站不该长 dashboard 脸)。settings drawer 精简保留 |
| 组件策略 | shadcn 只留交互骨架(Dialog / Popover / Tooltip / Drawer…);展示层组件(色卡、渐变条、场景卡、复制交互)全部自写,放 `src/components/showcase/` |
| 页面染色机制 | 复用现有 neutral 参数化:场景主色 hue 运行时写入 `--neutral-hue`(tint 适度抬高),强调色写入 `--primary` 一族 → 全站 token 自动跟随。**这是保留工程底座的核心理由** |

### 内容与语言

| 决策点 | 结论 |
|---|---|
| 语言 | **中文优先**,数据 schema 预留 `en` 字段,i18n 后置(Phase 3) |
| 内容组织 | 地区(蒙德/璃月/稻妻/须弥/枫丹/纳塔…)× 时间(晨/昏/昼/夜)× 情绪标签 |
| 注解生产 | AI 生成初稿 + 人工校对,注解与色卡同存于场景数据文件 |
| 冷启动量 | 10–15 个精选场景(蒙德、璃月为主)再上线,宁缺毋滥 |

### 技术架构

| 决策点 | 结论 |
|---|---|
| 架构 | **纯静态 SSG**:场景数据构建期读入,无数据库、无后端、无登录。开源部署零成本 |
| 场景数据 | `content/scenes/<id>.json`:元数据 + 色卡(OKLCH)+ 渐变停靠点 + 注解。数据文件是提取管线的**产物**,入库;截图原图**不入库** |
| 缩略图 | 仓库只存压缩 WebP(长边 ≤ 960、画质明显低于原图),定位是"取色来源参考",不提供原图/壁纸下载 |
| 提取管线 | 仓库内 CLI:`scripts/extract/`(Node + sharp)。两层输出:① OKLCH 空间 k-means 主色卡(感知均匀,比 RGB 聚类"像人挑的");② **方向性渐变重建**——沿主方向(天空竖向等)采样色带 → 拟合多停靠点 CSS 渐变。渐变重建是核心差异化,取色本身不是 |
| 色彩计算 | 引入 `culori`(OKLCH 转换/插值成熟、tree-shakable)。这是唯一计划新增的运行时依赖 |
| 导出格式 | CSS variables / Tailwind config / SVG gradient / Figma tokens JSON,一键复制 |
| OG 卡片 | `next/og`(satori)按场景生成:缩略图 + 色卡 + 渐变条 |
| 动画背景 | Phase 2 再定实现(倾向 canvas mesh gradient 自写 + `motion`,不引重依赖) |

## 里程碑

### M0 — 品牌与基调(小)

站名定稿;深灰蓝基底落进 `globals.css`(暗色为默认);替换 shell 为
顶部导航布局;选定标题字体。产出:一个有"脸"的空站。

### M1 — 提取管线 CLI(核心工程,可与 M0 并行)

`pnpm extract <截图路径>` → 输出场景 JSON + WebP 缩略图。
包含:OKLCH k-means、渐变方向检测与停靠点拟合、色卡 AA/AAA 对比度标注。
**M1 交付后即可开始积累截图素材,内容生产与站点开发解耦。**

### M2 — 数据 schema 定稿 + 3 个样例场景

用真实截图跑通管线,人工校对注解,验证 schema 覆盖度(地区/时间/情绪/
色卡/渐变/注解/缩略图外链)。

### M3 — 画廊 + 详情页 + 动态染色(产品主体)

- 画廊:网格 + 地区/时间筛选,场景卡 hover 预览渐变
- 详情页:渐变 hero、色卡条(OKLCH/HEX、点击复制)、导出面板、
  「为什么和谐」注解区
- 进入详情页时场景色染全站(`--neutral-hue` / `--primary` 运行时写入)
- 白底预览模式

### M4 — 传播与上线

OG 卡片、sitemap/SEO、合规声明、10–15 场景填充,部署 Vercel,
配 X 发布素材(对比图:场景 → 色卡 → 应用示例)。

### Phase 2(上线后)

渐变 Playground(scale/distortion/speed 可调的动画渐变,导出 CSS/React
组件——对标 Gradient Builder 类工具,但预设全部来自提瓦特)、收藏/合集、
搜索。展示组件反哺 yBlocks 成 showcase/gallery 模板。

### Phase 3

学习专栏(色彩理论 × 场景案例系列)、英文 i18n、社区场景投稿流程。

2. **站名确认**:Teyvat Palette 是工作名,M0 定稿。
